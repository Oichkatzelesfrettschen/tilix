# Roadmap

This document outlines the roadmap for the Tilix project.

## Short-Term Goals

The following are the short-term goals for the Tilix project:

* **Address the issue backlog**: There is a large backlog of open issues. The first priority is to go through the issue backlog and address as many issues as possible. This will include fixing bugs, adding new features, and improving the documentation.

* **Attract new maintainers**: The project is in need of new maintainers. The second priority is to attract new maintainers to the project. This will be done by creating a welcoming and supportive community, and by making it easy for new contributors to get involved.

* **Improve the documentation**: The documentation for Tilix needs to be improved. The third priority is to improve the documentation, including the architectural overview, the code documentation, and the user documentation.

* **Investigate the dependency issue**: The project is using a newer version of the GtkD library than is publicly available. The fourth priority is to investigate this issue and determine whether it is a problem.

## Long-Term Goals

The following are the long-term goals for the Tilix project:

* **Become the best terminal emulator for Linux**: The ultimate goal is to make Tilix the best terminal emulator for Linux. This will be done by continuing to add new features, improve the performance, and make the user interface as intuitive as possible.

* **Create a strong community**: A strong community is essential for the long-term success of any open source project. The goal is to create a strong community around Tilix that is welcoming and supportive of all users and contributors.

* **Ensure the long-term sustainability of the project**: The goal is to ensure the long-term sustainability of the project by attracting new maintainers, creating a strong community, and generating revenue.
